# Racketで学ぶ生命のゲーム

> **副題案**: 関数型プログラミング入門とコンウェイのライフゲーム  
> **目標**: 2026年9月11日 Kindle出版  
> **参照**: `my-racket/kyapon-knk-racket.org`（基礎問題50問）、Racket公式ドキュメント  
> **想定読者**: プログラミング初級～中級、Lisp/Scheme 系に興味がある人

---

## 目次ドラフト

### 序章　なぜRacketか——ゲームで学ぶ関数型

- 0.1 Racket とは（Scheme系・DrRacket）
- 0.2 ライフゲームを題材にする理由（状態・再帰・可視化）
- 0.3 本書の進め方と環境構築（Linux / macOS / Windows）
- 0.4 コードの取得：GitHub `my-racket` との関係

### 第1章　Racketの基礎——式と関数

- 1.1 DrRacket の起動と REPL
- 1.2 四則演算と `define`（問題1–10）
- 1.3 `print` と文字列（問題11–15）
- 1.4 真偽値と条件分岐 `if`（問題16–20）
- 1.5 リストの基礎 `list`, `car`, `cdr`（問題21–25）

### 第2章　再帰——Racketの核心

- 2.1 単純再帰の形（階乗・フィボナッチ）
- 2.2 `cond` とパターンマッチ的思考
- 2.3 末尾再帰と `let` / `let*` / `letrec`
- 2.4 高階関数 `map`, `filter`, `foldl`
- 2.5 ローカル変数と名前付き `let`

### 第3章　データ構造——グリッドを表現する

- 3.1 二次元リストとしての盤面
- 3.2 ベクター（`make-vector`）による高速化
- 3.3 構造体 `struct` でセルを定義
- 3.4 不変データと新世代の生成
- 3.5 モジュール分割の基本

### 第4章　ライフゲームのルール

- 4.1 コンウェイのルール説明（B3/S23）
- 4.2 隣接セル数を数える関数 `count-neighbors`
- 4.3 次世代を計算する `next-generation`
- 4.4 境界の扱い（トーラス面 vs 固定境界）
- 4.5 ユニットテストの書き方（`rackunit`）

### 第5章　描画と対話——盤面を見る

- 5.1 2htdp/image または ascii 表示
- 5.2 世代を進める REPL コマンド
- 5.3 有名パターン（グライダー・パルサー・ビーコン）
- 5.4 ファイルから初期配置を読み込む
- 5.5 GIF / 画像列のエクスポート（発展）

### 第6章　拡張ルール——変種ライフゲーム

- 6.1 HighLife（B36/S23）
- 6.2 ワイヤーワールド入門
- 6.3 ルール文字列のパーサ設計
- 6.4 ユーザー定義ルールの実験
- 6.5 ランダム初期配置と安定性

### 第7章　パフォーマンスと設計

- 7.1 プロファイリングの基礎
- 7.2 スパース表現（生きているセルのみ保持）
- 7.3 並列化の可能性（将来課題）
- 7.4 リファクタリング：関数型らしい設計
- 7.5 `typed/racket` へのヒント（任意）

### 第8章　ゲーム・オブ・ライフの数学と哲学

- 8.1 チューリング完全性の直感
- 8.2 自己組織化と複雑系
- 8.3 人工生命（ALife）への接続
- 8.4 プログラミング学習としての意義

### 付録
